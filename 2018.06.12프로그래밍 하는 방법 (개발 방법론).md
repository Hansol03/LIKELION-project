# 프로그래밍 하는 방법 (개발 방법론)

1. 일단 돌아가게 만든다. (엄청 무식한 방법이라도 상관 X)
2. 테스트를 한다.
3. 리팩토링(refactoring)을 한다.
   - 코드의 악취를 제거한다.
   - 악취1. 반복되는 코드(DRY<do not repeat yourself> 원칙을 지킨다.)
   - 악취2: 보기힘든 코드
   - 악취3: 비효율적인 코드

* 참고 - TDD(Test Driven Development) : 2-> 1 -> 3
